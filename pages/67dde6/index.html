maven的原理概念

# maven是什么？

没有深入学习之前，我总下意识的以为maven只是一个项目的依赖管理工具，但是深入学习之后我明白了自己的认知偏差。 maven的本质其实是一个项目管理工具，依赖管理只是maven的一部分核心功能，maven将项目开发和管理过程抽象成一个项目管理模型（POM），一个POM文件 对应着一个项目，通过POM的属性和配置的依赖管理项目需要用到的依赖和属性特征。

# maven的核心功能

  1. 项目构建
  2. 依赖管理

# maven目录

  1. bin: 核心运行文件，也就是mvn命令的源地址
  2. boot: 类加载器
  3. conf: 配置
  4. lib: jar包位置
  5. 其他的是许可证还有项目README文件

# maven工具的一些专业概念

  1. 仓库：用于存储所需jar包的资源，分maven官方管理的中央仓库和其他个人搭建的私服仓库
  2. 坐标：用于定位仓库中的jar包准确找到jar包，有一套固定的坐标规则
    • groupId：定义当前maven项目所属组织名称（通常是域名反写，例如：org.mybatis）
    • artifactId：定义当前maven项目名称（通常是模块名称，例如：com等）
    • version：当前版本号
    • packaging：定义项目的打包方式

# maven常用命令

  1. mvn clean 清理项目
  2. mvn package 打包
  3. mvn install 在本地生成仓库的安装包，以供其他项目使用
  4. mvn test 对项目进行运行测试
  5. mvn compile 编译项目源代码
