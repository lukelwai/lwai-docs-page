# MySQL的锁

MySQL中有三种级别的锁，分别是表级锁、行级锁、页级锁

# innodb数据引擎中的锁

innodb数据引擎中表级锁分三种，意向共享锁（IS）、意向排他锁（IX）、自增锁（AUTO-INC Locks）

  1. 意向共享锁(IS)
    多个事务可以一起读，共享锁之间不排斥（也就是可以继续添加共享锁），但是阻塞意向排他锁
  2. 意向排他锁(IX)
    只允许获得排他锁的事务更新数据，阻止其他事务获取相同数据集的共享、排他锁
  3. 自增锁
    特殊表锁，自增长计数器通过获取该锁，来获取子增长计数器最大的计数值

# 什么是关系型数据库？

关系型数据库是建立在关系模型基础上的数据库，也可以简单理解为二维数据库，表的格式例如excel，分行和列。

# 1. 常用的关系型数据库

MySQL、Oracle、SQL server等。

# 2.非关系型数据库

非关系型数据库可简称为：简称NOSQL,非关系型数据库的存储格式有key,value形式、文档形式、图片形式等， 常见的非关系型数据库有：redis、MongoDB、elasticsearch,严格来说，elasticsearch不是一个数据库而是一个搜索引擎。

# 关系模型

# 1. 什么是关系？

关系是一张二维表，表示数据的逻辑结构。表中一行表示一个记录（元组）， 一列表示一个属性，属性的取值范围表示域

# 2. 什么是关系模型？

关系模型指的是二维表结构表示的实体关系，用键表示实体间联系的数据模型。 关系模型由关系数据结构、关系操作、关系完整性约束组成。

# 数据的三级模式

数据的三级模式为：外模式、模式、内模式

# 外模式

外模式也称为用户模式，他是数据库用户能够看见和使用的局部数据的逻辑结构和特征描述。 是数据库用户的数据视图，一个数据库可以有多个外模式

# 模式

模式也成为逻辑模式和概念模式，是数据库中全体数据的逻辑上结构和特征描述，是所有用户的公共数据视图。 一个数据库只有一个模式，模式位于三级结构的中间层

# 内模式

内模式也成为内存模式，一个数据库只有一个内存模式，它是数据库物理结构和存储方式的描述，是数据在数据库内部的表示方式
