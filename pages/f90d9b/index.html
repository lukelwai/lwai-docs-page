考试知识笔记-数据库系统

考点：

  1. 数据库体系结构
  2. 三级模式结构
  3. 数据仓库

# 1. 数据库体系结构

  1. 集中式数据库
    • 数据是集中的
    • 数据管理是集中的
  2. C/S结构
    • 客户端负责数据表示服务
    • 服务器主要负责数据库服务
    • ODBC、JDBC
  3. 分布式数据库
    • 物理上分布、逻辑上集中
    • 数据独立性。除了逻辑独立性和物理独立性之外还存在数据独立性
    • 集中和自治共享结合的控制结构：
    • 适当增加数据冗余：在不同的场地增加统一数据的副本，可以提高系统的可用性和可靠性。
    • 提高系统的可用性，就是当其中一个节点出现问题之后，可以从其他的节点中将数据进行恢复，是系统不瘫痪就是指的可用性
    • 分片透明：用户不用关心数据如何分片（只要涉及到分块、分片基本都是分片透明）
    • 复制透明：用户不用关心数据在各个节点的复制情况
    • 位置透明：用户不用关心所操作的数据在何处
    • 逻辑透明：用户不用关心局部使用的数据模型
  4. 并行数据库

# 2. 三级模式结构

外模式和概念模式有一定的映射（逻辑独立性），概念模式和内模式之间也有一定的映射（物理独立性），修改二者的映射关系就可以完成间接调用

  1. 内模式（文件级）：也叫内存模式
  2. 概念模式（表级）：增查删改，操作的是表级
  3. 外模式（视图级）：也叫用户模式，也就是用户直接操作的一级，一般只可查不可改

# 3. 数据仓库

OLAP: 联机分析，着重后期分析，OLTP：联机事务系统，着重链接事务处理 特点：

  1. 面向主题：数据按主题组织
  2. 集成的：消除了源数据中的不一致性
  3. 相对稳定的：主要进行查询操作，删改较少
  4. 反映历史变化：记录企业从过去某一时刻到当前各个阶段的信息。可对发展历史和未来趋势做定量分析和预测

阶段：

  1. 阶段一：数据预处理（ETL），例如抽取、清理、清洗等等
  2. 阶段二：数据仓库存储
  3. 阶段三：数据分析
