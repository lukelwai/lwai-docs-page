数据中台理解

# 什么是数据中台？

数据中台是一套可持续“让企业的数据用起来”的机制，简单来说也可以理解成一套可持续数据集成、加工管理、服务的平台。

# 特点

  1. 数据中台具备数据汇聚整合、数据提纯加工、数据服务可视化、数据价值变现核心能力。
  2. 数据中台的核心就是实现公共计算逻辑下沉，实现数据复用，提供给接口使用。
  3. 数据中台不是某一个单一的产品或者某个技术。本质上讲数据中台就是从数据中发现价值，赋能业务数据管理机制。

# 拆解

数据中台前面的数据二字很好理解，就是我们平常说的理解的数据，那什么是”中台“呢？刚开始理解开发的时候我们会很容易接触到两个词汇”前端“、”后端“， 有时候会我们也会说成”前台“、”后台“，那”中台“呢？我们可以先理解一下”前台“和”后台“的概念

  1. 前台是系统的前端平台，是直接与终端用户进行交互的应用层。
  2. 后台是指系统的后端平台，终端用户是感知不到他的存在的。后台的价值是存储和计算企业的核心数据。

用户需求的变化决定了前台系统需要快速迭代响应用户需求，而前端的变化需要后端的变化来支撑，因此这就对后台的快速应变产生了要求。 而后台设立之初核心目的并不是服务于前台，而是提升后端数据的安全及系统的管理效率。因此基于“前台+后台”的架构发展演变，催生了”中台“的概念， 用于支撑”前台“的快速迭代需求和服务于”后台“的数据需求

# 中台

中台又包含业务中台、技术中台、数据中台。

  1. 业务中台，实际上是后台的延伸，实现资源或能力的极度共享，沉淀出可以“被共享的能力”，要实现能力的沉淀。
  2. 数据中台，应该被称之为“业务中台的数据化”，其让资源和能力在极度共享后形成数据汇集，并基于算法进行智能决策，以支持业务中台和组织中台的运作。
  3. 技术中台，负责基础服务、基础组件、基础平台、存储体系、云平台、运维相关等技术支撑。

# 数据中台的价值

  1. 提升数据应用能力，将复杂的数据转化成高质量的可读、可用数据，提供个性化和智能化的数据服务
  2. 打破数据应用屏障，将用技术将数据转化成业务人员也可以理解读取的数据，打破技术人员和业务人员的隔阂，让业务人员直接进行数据操作和读取
  3. 打破数据孤岛，盘活全量数据，降低数据使用门槛
  4. 支持跨主题域访问数据，可以从全局、全域的数据角度出发，访问和分析数据
  5. 数据快速复用，统合企业应用，实现数据资源的共享和复用
